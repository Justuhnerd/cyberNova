Here are solid justifications you can use in your portfolio report for the gold tooltip design decision:
For your methodology/design decisions section, frame it like this:
The gold dashed underline on technical terminology was implemented as a contextual glossary mechanism. The decision was grounded in Nielsen's Usability Heuristic 6 — Recognition Over Recall — which states that users should not need to memorise information to use an interface. Security terminology such as "EDR," "UEBA," "RaaS," and "POPIA" carries significant meaning for expert users but creates comprehension barriers for the SME and non-technical government staff that CyberNova explicitly serves. Rather than directing users away from the page to a separate glossary, inline tooltips preserve reading flow while surfacing definitions exactly when and where the user encounters the unfamiliar term.
The gold colour was not an arbitrary aesthetic choice. It directly inherits from CyberNova's established brand accent, creating a consistent visual language: gold already signals interactivity and importance across the site (CTAs, active nav states, the admin badge). Applying it to tooltip triggers extends that convention — users who have learned that gold means "pay attention" naturally recognise gold-underlined terms as interactive.
The dashed rather than solid underline distinguishes these from standard hyperlinks, avoiding the false affordance of suggesting navigation when the interaction is a popup definition.
One-liner version for a table or rationale column:
"Gold dashed underline aligns with the site's established gold-equals-interactive convention and implements Nielsen's Heuristic 6 by surfacing domain-specific definitions in context, reducing cognitive load for non-technical users without interrupting reading flow."
That gives you a design principle (Nielsen H6), a brand consistency argument, and a user group justification — three separate angles the examiner can credit.

function editItem(id) {
  const item = CyberNovaDB.getContent().find(c => c.id === id);
  if (!item) return;
  const newTitle = prompt('Edit title:', item.title);
  if (newTitle && newTitle.trim()) {
    CyberNovaDB.updateContent(id, { title: newTitle.trim() });
    renderContentTable();
  }
}


function doArchive(id) {
  const item = CyberNovaDB.getContent().find(c => c.id === id);
  if (!item) return;
  if (item.status === 'archived') {
    CyberNovaDB.updateContent(id, { status: 'published' });
    showUndo(`"${item.title}" restored.`);
  } else {
    CyberNovaDB.archiveContent(id);
    lastArchived = id;
    showUndo(`"${item.title}" archived.`);
  }
  renderContentTable();
}
